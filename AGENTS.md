# Правила роботи над CRM

## 1. Робота з невідомою інформацією

- Якщо інформації бракує або щось невідомо, потрібно прямо повідомити про це й поставити користувачу уточнювальне запитання.
- Заборонено вигадувати факти, вимоги, бізнес-логіку, технічні обмеження або контекст.
- Припущення не можна подавати як підтверджену інформацію.

## 2. Статус вимог і рішень

Кожну вимогу або технічне рішення потрібно відносити до одного зі статусів:

- **Підтверджено** — користувач явно погодив вимогу або рішення.
- **Запропоновано** — варіант запропонований для обговорення, але ще не погоджений.
- **Відкрите питання** — інформації недостатньо; перед реалізацією потрібно отримати відповідь користувача.

Не можна реалізовувати запропоновані або відкриті вимоги як підтверджені без погодження користувача.

## 3. Безкоштовні засоби розробки

- Усі мови, фреймворки, бібліотеки, інструменти та сервіси, які використовуються під час розробки, мають бути безкоштовними.
- Не можна додавати залежності від платних ліцензій, підписок, API або функцій без окремого дозволу користувача.
- Це обмеження стосується етапу розробки. Умови майбутнього зовнішнього хостингу ще не визначені.

## 4. Середовище запуску

- На поточному етапі CRM розробляється та запускається локально на комп'ютері користувача.

## 5. Актуальні проєктні рішення

- Backend: Python, Django, Django REST Framework.
- Frontend: JavaScript, Vue 3.
- База даних: MySQL.

Детальні вимоги зберігаються у `PROJECT_REQUIREMENTS.md`.
